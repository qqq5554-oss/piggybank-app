# 🐷 小小存錢筒 — 後續開發清單

給 Claude Code 接手用。專案架構：React + Vite + Vercel Serverless Functions（`/api`）+ Neon Postgres。

## 🔴 優先：先確認部署真的能動

- [ ] 打開 Vercel 部署後的網址，確認首頁能看到「小安」「小美」兩個帳戶卡片
- [ ] 點進小孩帳戶，確認能看到餘額、家事清單
- [ ] 測試「完成家事」→ 家長模式（PIN: 0000）→「審核」分頁能看到申請並核准，餘額有正確加總
- [ ] 測試家長模式「管理帳戶」「家事清單」「設定」三個分頁的加減錢、改密碼功能都正常寫入
- [ ] 確認手機瀏覽器打開時排版正常（不是只在電腦測試）

## ⚙️ 技術債 / 注意事項

- [ ] 本機測試流程要用 `vercel dev`（不是 `npm run dev`），因為要測到 `/api`
- [ ] 之後如果要加新的家長操作，記得都要走 `api/action.js` 的 PIN 驗證機制（`PARENT_ACTIONS` 集合），不要漏掉安全檢查
- [ ] 家長 PIN 驗證成功後存在瀏覽器記憶體（`App.jsx` 的 `parentPin` state），重新整理頁面就要重新輸入 —— 這是刻意設計、不需要改成永久登入
- [ ] `DATABASE_URL` 只能存在 Vercel 的伺服器端環境變數，絕對不能加 `VITE_` 前綴（會被打包進前端暴露出去）
