## ✅ 已完成：Phase 1（生活責任 / 特殊任務 / 責任值 / 違規紀錄）

家長要求的完整功能清單裡，先做完不需要排程的 5 項：

- 🏠 **生活責任**：`responsibilities` + `responsibility_logs` 資料表，小孩每天打卡，`utils/format.js` 的 `computeStreak()` 算連續完成天數，完成加「責任值」（跟金錢分開的獨立帳本 `character_point_logs`）
- 🎯 **特殊任務**：`missions` 資料表（open → pending → done），小孩送出完成申請、家長在「審核」分頁核准，核准後轉一筆 `transactions` 收入
- ⭐ **責任值**：`kids.character_points` 欄位 + `character_point_logs` 紀錄，家長可在「管理帳戶」手動加減並附原因
- ❌ **違規紀錄**：`violations` 資料表，家長登記時可同時扣錢（走 `transactions` type='penalty'）、扣責任值、記錄禁止的權利（純文字備註）
- 🎁 **人生事件**：沒有另外建資料表，直接沿用既有的「加減錢」（`adjust_balance`）就能達到效果（例如備註打「🧧 過年」），不需要額外開發

家長後台新增了「生活責任」「特殊任務」兩個管理分頁，「審核」分頁合併顯示待審核家事與任務。

## ✅ 已完成：Phase 2（固定零用錢 / 固定支出 / 存錢利息）

- 💸 **固定零用錢**：`allowance_rules` 資料表，每週（指定星期幾）或每月（指定幾號）自動入帳
- 🏠 **固定支出**：`expense_rules` 資料表，每月指定幾號自動扣款
- 📈 **存錢利息**：`kids.interest_rate` 欄位，每月 1 號依當時餘額結算利息
- 全部由 `api/cron.js` 執行，`vercel.json` 設定每天 UTC 00:00 觸發一次（`0 0 * * *`）
- 用 `scheduled_run_logs` 表防止同一天重複執行；月結日超過當月天數（例如設定 31 號但 2 月只有 28 天）會自動改用月底那天，已用單元測試驗證含閏年
- 家長後台新增「定期收支」分頁管理這三項

## ⚙️ 技術債 / 注意事項

- [ ] 本機測試流程要用 `vercel dev`（不是 `npm run dev`），因為要測到 `/api`
- [ ] 之後如果要加新的家長操作，記得都要走 `api/action.js` 的 PIN 驗證機制（`PARENT_ACTIONS` 集合），不要漏掉安全檢查
- [ ] 家長 PIN 驗證成功後存在瀏覽器記憶體（`App.jsx` 的 `parentPin` state），重新整理頁面就要重新輸入 —— 這是刻意設計、不需要改成永久登入
- [ ] `DATABASE_URL` 只能存在 Vercel 的伺服器端環境變數，絕對不能加 `VITE_` 前綴（會被打包進前端暴露出去）
